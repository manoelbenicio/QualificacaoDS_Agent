2. Detalhamento dos Componentes

Descrição técnica de cada camada e serviço

▼
🪣

Frontend — S3 + CloudFront

AWS
  • S3 Bucket — Hospedagem estática da aplicação SPA
  • CloudFront — CDN para distribuição global com HTTPS
  • Framework — React (SPA com roteamento client-side)
  • Autenticação — AWS Cognito (OAuth 2.0 / OIDC)
  • Build — Vite + deploy via CI/CD (GitHub Actions)
🔀

API Gateway — REST API

AWS
  • Tipo — API Gateway REST (regional)
  • Autorização — Cognito Authorizer (JWT tokens)
  • Throttling — Rate limiting por API key / user
  • CORS — Habilitado para domínio do frontend
  • Versionamento — Stages (dev, staging, prod)
  • Integração — Lambda Proxy Integration
λ

Lambda Functions

AWS
  • Runtime — Python 3.12
  • Memória — 512MB–1024MB (agentes)
  • Timeout — 60s (CRUD) / 300s (agentes)
  • Packaging — Container Image (ECR) para agentes, ZIP para CRUD
  • Concorrência — On-Demand (auto-scaling)
  • Layers — Shared: boto3, google-adk, langchain
🤖

Google ADK — Agentes de IA

Google
  • Framework — Google Agent Development Kit (ADK)
  • Sessão — Persistida em DynamoDB (session_id → state)
  • Multi-turn — Chat com refinamento via sessão contínua
  • Tools — Custom tools para RAG (SharePoint), cálculos
  • Models — Claude 3.5 Sonnet (Ag. 1/3), GPT-4o (Ag. 2)
  • Callbacks — Logging, métricas e token tracking
🗄️

DynamoDB — Persistência

AWS
  • Sessions Table — PK: session_id | Sessões ADK com TTL
  • Ofertas Table — PK: offer_id, SK: step | Status, outputs
  • Config Table — PK: agent_id | Prompts, pesos, KBs
  • Users Table — PK: user_id | Perfil, permissões
  • Modo — On-Demand Capacity (auto-scaling)
  • Backup — Point-in-Time Recovery habilitado
📁

SharePoint — Files & Knowledge

Microsoft
  • Acesso — Microsoft Graph API (OAuth 2.0)
  • Site — SharePoint dedicado "AgentificacaoOfertas"
  • Libraries — /Uploads, /Outputs, /KB-* (6 KBs)
  • Indexação KB — Pipeline de embedding (chunking → vetorização)
  • Permissões — RBAC via Azure AD
  • Retenção — Políticas corporativas Indra
λ

7. Decisões de Arquitetura & Notas

Premissas, trade-offs e decisões técnicas

▼

Decisões Chave

  • Google ADK vs. LangChain: ADK oferece gerenciamento nativo de sessão, multi-turn e integração multi-model sem abstração adicional
  • DynamoDB para sessões ADK: Session store com TTL automático, baixa latência e serverless
  • SharePoint vs. S3 para arquivos: SharePoint é o padrão corporativo Indra — mantém governança e permissões
  • Lambda Container vs. ZIP: Agentes usam Container Image (ECR) para deps pesadas; CRUDs usam ZIP para cold start rápido
  • Multi-model (Claude + GPT-4o): Cada agente usa o modelo mais adequado
  • Cognito + Azure AD: SSO federado com Azure AD corporativo via SAML/OIDC

Premissas & Restrições

  • AWS é o cloud provider padrão para novos workloads de IA da Indra Minsait Brasil
  • SharePoint Online disponível com M365 corporativo para todos os usuários
  • APIs de LLM via API keys em AWS Secrets Manager
  • Volume: ~50 ofertas/mês com picos de até 200 execuções/dia
  • Latência aceitável: até 30 segundos para resposta completa
  • Custo estimado AWS (infra): ~$500-800/mês; LLM tokens: variável
