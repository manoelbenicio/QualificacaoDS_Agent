Componentes do Pipeline

server.py — API HTTP Flask (porta 8006)
extracao.py — Extrator multi-formato (11 tipos)
validacao.py — Validador de inputs (enums)
acervo.py — Catálogo de 2.042 documentos
maestro.py — Orquestrador SA1→SA5
adk_robo.py — Motor ADK (sem API)
trace_logger.py — Audit trail completo
sa1-sa5.py — Configs p/ modo LLM

Baixo Nível — Fluxo de Dados Interno

1

Upload & Validação

server.py /api/qualificar recebe multipart/form-data

  • Valida extensão (.pdf, .docx, .eml, .xlsx, .txt, etc.)
  • Salva em /tmp
  • Parseia JSON dados do formulário
  • ValidadorInput.validar() — checa enums de setor/tipo
2

Extração Documental

ExtratorDocumental.extrair()

  • Router por extensão → parser específico
  • PDF: pdfplumber → texto por página
  • EML: email.parser → body + attachments
  • DOCX: python-docx → parágrafos + tabelas
  • Retorna texto_consolidado
3

SA1 — Extrator Estruturado

adk_robo.sa1_extrair()

  • Tokeniza texto (regex word boundary)
  • Detecta tecnologias (48 keywords com pesos)
  • Parseia valores monetários (R$, USD)
  • Identifica escopo principal (10 categorias)
  • Estima complexidade: BAIXA → MUITO_ALTA
4

Acervo DS — Confronto com KB

adk_robo._confrontar_acervo()

  • Busca em 2.042 documentos do catalog.json
  • Classifica por: Atestados, Cases, Capacidades, Propostas, Soluções, Templates
  • Calcula cobertura_score (0-10)
  • Retorna top 30 docs mais relevantes
5

SA2+SA3+SA4 — Análise Paralela

SA2 Técnico
Tech score + aderência DS + dimensionamento FTEs + perfis + gaps
SA3 Riscos
26 patterns → severidade ALTO/MEDIO/BAIXO + mitigações + risk_score
SA4 Comercial
8 Faróis (peso %) + Best Story Wins (4 pilares) + score ponderado
6

SA5 — Decisor Estratégico

adk_robo.sa5_decidir()

  • Score ≥ 7.0 + risk ≤ 4 → GO
  • Score < 4.0 ou ≥4 críticos → NO_GO
  • Demais → GO_CONDICIONADO + condições
  • Gera: farois, BSW, dimensionamento, riscos, next_steps
