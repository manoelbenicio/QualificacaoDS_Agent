1. Abordagem: Core First

Filosofia de implementação — entregar valor funcional rapidamente

▼

Princípio: Implementar o pipeline de agentes funcional antes de features secundárias

A estratégia Core First prioriza as funcionalidades essenciais: Login + Pipeline de 3 Agentes (Qualificação → Kickoff S2S → Ballpark). Features complementares (gestão de KBs, dashboard, configurações avançadas) são postergadas para fases posteriores.

Essa abordagem reduz riscos técnicos (valida ADK + LLM + SharePoint cedo), permite feedback dos pilotos, e evita retrabalho em features que podem mudar após validação do core.

⚡ Objetivo: MVP Core funcional em 6 semanas
📋

Por que Core First?

  • Reduz risco técnico: Valida ADK + LLM + SharePoint nas primeiras semanas
